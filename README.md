# ORLtrack

ORLtrack - это ASP.NET Core MVC-платформа для преподавателя, в которой можно:

- добавлять учеников
- хранить для каждого ученика индивидуальную ставку за занятие
- пополнять баланс
- отмечать проведенные занятия с автоматическим списанием
- фиксировать пропуски без списания баланса
- смотреть обзор, карточки учеников и аналитику
- получать AI-анализ риска оттока ученика через российскую нейросеть GigaChat

## Стек

- .NET 8
- ASP.NET Core MVC
- Entity Framework Core
- PostgreSQL
- GigaChat API

## Структура

- `plt/` - основное веб-приложение
- `plt/Controllers/` - контроллеры
- `plt/Models/Model/` - EF-модели и `DbContext`
- `plt/Models/ViewModel/` - view models
- `plt/Views/` - Razor-представления
- `plt/Services/Ai/` - сервис AI-аналитики через GigaChat

## Основные возможности

### Ученики

Для каждого ученика в системе хранится:

- имя и контакты
- баланс
- ставка за занятие
- число проведенных занятий
- история оплат
- история занятий и пропусков

### Занятия

Поддерживаются два сценария:

1. Проведенное занятие  
   Баланс уменьшается на ставку ученика, обновляются метрики и создается запись в истории.

2. Пропуск  
   Создается запись в истории, но баланс не списывается.

### Аналитика

В проекте реализована аналитика риска ухода ученика:

- локальный расчет `risk score` по поведению ученика
- сигналы риска в карточке ученика
- отдельная страница аналитики
- объяснение и рекомендация от GigaChat

Факторы риска, которые учитываются сейчас:

- сколько дней прошло с последнего занятия
- сколько дней прошло с последнего пополнения
- есть ли недостаточный баланс
- были ли пропуски в недавней истории

## Запуск проекта

### 1. Требования

Нужно установить:

- .NET SDK 8
- PostgreSQL

### 2. Настройка базы данных

По умолчанию проект использует базу:

`Platforma`

Строка подключения находится в:

`plt/appsettings.json`

Пример:

```json
"ConnectionStrings": {
  "DefaultConnection": "Host=localhost;Port=5432;Database=Platforma;Username=postgres;Password=YOUR_PASSWORD;"
}
```

При старте приложение вызывает `Database.Migrate()`, поэтому миграции применяются автоматически.

### 3. Запуск

Из корня проекта:

```bash
dotnet build movieRecom.sln
dotnet run --project .\plt\movieRecom.csproj
```

## Настройка GigaChat

Конфигурация находится в:

`plt/appsettings.json`

Секция:

```json
"GigaChat": {
  "Enabled": true,
  "AuthorizationKey": "YOUR_AUTHORIZATION_KEY",
  "Scope": "GIGACHAT_API_PERS",
  "Model": "GigaChat-2-Pro",
  "OAuthUrl": "https://ngw.devices.sberbank.ru:9443/api/v2/oauth",
  "ChatUrl": "https://gigachat.devices.sberbank.ru/api/v1/chat/completions",
  "IgnoreSslCertificateErrors": false,
  "MaxStudentsPerRequest": 3
}
```

### Как получить доступ

1. Получите `Authorization key` в кабинете GigaChat API.
2. Вставьте его в `AuthorizationKey`.
3. Установите `Enabled: true`.

После этого ORLtrack будет:

- запрашивать access token у GigaChat
- отправлять в модель данные по самым рискованным ученикам
- показывать на странице аналитики AI-объяснение и рекомендацию

Если GigaChat отключен или недоступен, система использует локальный fallback-анализ и интерфейс продолжает работать.

## Важное замечание по безопасности

Не храните реальные ключи API в публичном репозитории.

Для продакшена лучше вынести:

- строку подключения к БД
- `GigaChat.AuthorizationKey`

в:

- User Secrets
- переменные окружения
- защищенный конфиг на сервере
